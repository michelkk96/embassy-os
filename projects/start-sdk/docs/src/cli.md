# CLI Reference

The `start-cli` tool handles both building packages and managing registries. The full command reference lives in the [start-cli Reference](/start-os/cli-reference.html) — these sections are most relevant to service developers:

- [S9PK Packaging](/start-os/cli-reference.html#s9pk-packaging) — build, inspect, edit, and publish `.s9pk` packages
- [Registry](/start-os/cli-reference.html#registry) — manage packages, categories, signers, and OS versions on a registry

Starting out, the two commands you'll use first are [`s9pk init-workspace`](/start-os/cli-reference.html#start-cli-s9pk-init-workspace-path) (create a workspace) and [`s9pk init-package`](/start-os/cli-reference.html#start-cli-s9pk-init-package-name) (scaffold a package) — see [Environment Setup](./environment-setup.md) and [Quick Start](./quick-start.md).
